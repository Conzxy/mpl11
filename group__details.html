Macro expanding to a struct when generating the documentation and to an alias otherwise.

This is required because Doxygen handles template aliases (and aliases in general) poorly (or maybe I just can't get them to work).

#define BOOST_MPL11_ENABLE_ASSERTIONS

When defined, optional assertions and consistency checks are performed by the library.

Since this can have a negative impact on compile-time performance, it is disabled by default. However, it is strongly recommended to enable this when debugging template code.
